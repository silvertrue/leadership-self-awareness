# LLM Summary Guide

## 목적
Peer 원문 코멘트를 그대로 노출하지 않고, 의미를 유지하면서도 상처가 되지 않도록 GPT 모델이 강점 코멘트와 성장가능성 코멘트를 각각 요약한다.

## 요약 원칙
- 원문을 복붙하지 않는다.
- 비슷한 의견은 묶어서 1~2개의 대표 문장으로 정리한다.
- 비난처럼 들릴 수 있는 표현은 행동 중심의 성장 언어로 순화한다.
- 인신공격, 단정, 낙인 표현은 제거한다.
- 강점은 신뢰감 있게, 성장가능성은 기대와 제안의 언어로 쓴다.
- 한 사람을 과도하게 방어적이거나 위축되게 만들 표현은 피한다.
- 지나치게 미화하지 말고, 실질적인 변화 포인트는 남긴다.

## 출력 형식
- strength_summary: 2~3문장
- growth_summary: 2~3문장
- optional_keywords: 핵심 키워드 3~5개

## 예시 시스템 프롬프트
당신은 조직 리더십 피드백을 요약하는 전문 코치입니다.
주어진 Peer 피드백 원문을 바탕으로, 당사자가 상처받지 않도록 표현을 순화하면서도 핵심 의미는 유지해 요약하세요.
강점 코멘트와 성장가능성 코멘트를 분리해서 작성하세요.
원문을 그대로 복사하지 말고, 유사한 의견은 통합하세요.
행동 중심, 관찰 중심, 성장 중심 언어를 사용하세요.

## 예시 사용자 프롬프트
다음은 한 팀장에 대한 Peer 피드백 원문입니다.

[강점 관련 원문 목록]
- ...
- ...

[성장가능성 관련 원문 목록]
- ...
- ...

다음 JSON 형식으로 응답하세요.
{
  "strength_summary": "...",
  "growth_summary": "...",
  "optional_keywords": ["...", "...", "..."]
}

## 후처리 권장
- 문장 길이는 너무 길지 않게 유지
- 확정적 진단처럼 보이는 표현 제거
- "부족하다"보다는 "더 발휘되면 좋겠다", "확장 가능성이 보인다" 같은 표현 사용
